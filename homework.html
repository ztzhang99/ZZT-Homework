Homework of Introduction to Software Engineering -- hw7

  1. Program with machine language according to the following c.
    int_8 a = 1;
    int_8 c = a + 3;
    1. Write your assembly code & machine code
      Answer: (MIPS)
      • Assembly code:
          .data
  a:  .byte
  c:  .byte

  .text
  main:
    la  $s0, a
    addi  $t0, $0, 1
    sw  $t0, 0($s0)
    la  $s1, c
    addi  $t0, $t0, 3
    sw  $t0, 0($s1)

    li  $v0, 10
    syscall
      • Machine code:
          001000 01000 00000 0000000000000001 #addi
  101011 01000 10000 0000000000000000 #sw
  001000 01000 01000 0000000000000011 #addi
  101011 01000 01001 0000000000000000 #sw
    2. Explain machine code execution with the fetch-decode-execute cycle
      Answer:
      Fetch:
      Fetch a line of code from the memory, and store it in IR. Then, add 4(or sth else) to PC.
      Decode:
      Set different control signal depending on the opcode, and load value from registers(if needed).
      Execute:
      Do different operations depending on the control signal.
    3. Explain functions about IR, PC, ACC registers in a CPU
      Answer:
      IR:
      Store a line of code, which is loaded from memory.
      PC:
      Record the address of a line of code, which is going to be loaded a clock time later.
      ACC:
      Temporarily store the result of an add operation. Without ACC, we must store the result to memory and load it back immediately, which is really a waste of time.
    4. Explain physical meaning about vars a & c in a machine
      Answer:
      Both a and c are addresses. Through these addresses, we are able to store and retrieve data to and from memory.
  2. 简答题
    1. What are stored in memory?
      Answer:
      Generally speaking, data. Instructions are also seen as data.
    2. Can a data or a instruction stored in the same place?
      Answer:
      No. According to principle of locality, it is not efficient to do so.
    3. Explain Instruction Format with example instructions.
      Answer:
      001000 01000 00000 0000000000000001 #addi
      31-26: The opcode of "addi". 25-21: 8, the sign of register $t0. 20-16: 0, the sign of register $zero. 15-0: 1, an immediate number, which will be added to $t0.
  3. 解释以下词汇
    1. 汇编语言(Assembly Language)
      Explaination:
      A low level language. After transformed into machine language by assembler, the code can be directly understood by CPU.
    2. 编译(Compiler)
      Explaination:
      A process to transform high level language into machine language, which is done by compiler.
    3. 命令式语言(Imperative programming)
      Explaination:
      A kind of languages like C & C++. These languages are based on the Von Neumann organization.
    4. 函数编程语言(Functional programming)
      Explaination:
      A kind of languages like Haskell. There is no variable or side effect in these languages. Compiler can do more optimization in these languages than in imperative languages.
    5. 过程式编程(Procedural programming)
      Explaination:
      The only way of programming which I have programmed.
